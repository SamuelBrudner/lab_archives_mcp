# JORS Submission Checklist

## Repository
- [x] `README.md` is up to date and includes installation instructions.
- [x] `LICENSE` is present (OSI-approved).
- [x] `CONTRIBUTING.md` and `CODE_OF_CONDUCT.md` are present.
- [x] `CITATION.cff` is present and valid.
- [x] Code is modular and well-documented.
- [x] Tests are passing and cover core functionality.

## Metapaper
- [x] `jors_metapaper.md` is complete.
- [x] Author ORCID IDs are correct.
- [x] Affiliations are correct.
- [x] References are in `paper.bib`.

## Submission
- [x] Create a release tag (e.g., `v0.3.0`) corresponding to the submission.
- [ ] Archive the release on Zenodo (or similar) to get a DOI.
- [ ] Submit via the JORS portal.
