# Linked-data provenance export

`labarchives_mcp` maintains a project-scoped NetworkX graph in local state. In `v0.4.0`, that graph can be exported as JSON-LD aligned to PROV-O and schema.org so the same project memory can be inspected outside the MCP server without committing the server itself to a triple store or SPARQL stack.

## Scope

The export is intentionally narrow:

- It serializes one saved project context at a time.
- It supports both legacy project graphs and the new upload-provenance subgraph.
- It does not ingest external RDF/JSON-LD or provide a query endpoint.

## Node mapping

| Graph node type | JSON-LD types |
| --- | --- |
| `project` | `prov:Collection`, `schema:Dataset` |
| `notebook` | `prov:Collection`, `schema:Collection` |
| `page` | `prov:Entity`, `schema:CreativeWork` |
| `finding` | `prov:Entity`, `schema:CreativeWork` |
| `artifact` | `prov:Entity`, `schema:DigitalDocument` |
| `activity` | `prov:Activity` |
| `user` | `prov:Agent`, `schema:Person` |
| `software_agent` | `prov:Agent`, `prov:SoftwareAgent`, `schema:SoftwareApplication` |

## Edge mapping

| Graph edge relation | JSON-LD / RDF meaning |
| --- | --- |
| `uses_notebook`, `contains`, `visited`, `tracked`, `discovered` | collection membership via `prov:hadMember` |
| `contains_artifact` | artifact is nested under a page via `schema:isPartOf` |
| `evidence_from` | finding derives from a page via `prov:wasDerivedFrom` |
| `was_generated_by` | entity generated by an activity via `prov:wasGeneratedBy` |
| `used` | activity consumed another entity via `prov:used` |
| `was_attributed_to` | entity attributed to an agent via `prov:wasAttributedTo` |
| `was_associated_with` | activity associated with an agent via `prov:wasAssociatedWith` |
| `was_informed_by` | activity informed by another activity via `prov:wasInformedBy` |
| anything else | emitted as `labmcp:relatedTo` |

## Upload provenance

Successful `upload_to_labarchives(...)` calls enrich the active project graph with a concrete provenance subgraph:

- the created page remains a `page` entity
- the uploaded content is represented as an `artifact` entity
- the upload is represented as an `activity`
- the authenticated LabArchives user is represented as a `user` agent
- the MCP server is represented as a `software_agent`

The activity node carries the metadata already validated by `ProvenanceMetadata`: Git commit SHA, Git branch, repository URL, dirty state, code version, Python version, dependencies, OS, hostname, and server version.

## Interfaces

- MCP tool: `export_provenance_jsonld(project_id)`
- CLI: `labarchives-mcp export-provenance --project <id> --output graph.jsonld [--state-dir <path>]`
- Module form: `python -m labarchives_mcp export-provenance --project <id> --output graph.jsonld`

## Non-goals

- No ontology redesign beyond PROV-O + schema.org + a small `labmcp:` namespace
- No triplestore or SPARQL endpoint
- No attempt to stabilize URIs across different local state directories
